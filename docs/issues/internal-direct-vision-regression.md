# Issue：internal direct 链路疑似退化为“气泡 OCR + 文本翻译”

- 状态：open
- 记录时间：2026-05-17
- 关联项目：manga-translate-agent
- 关联模块：internal/direct 链路、Vision Stage、QA Stage

## 问题描述
当前怀疑 internal direct 链路效果差，不是单纯模型翻译能力不足，而是 **Vision Stage 的 prompt / 任务定义退化**：模型虽然看到了页面，但实际主要在读气泡文字，没有充分利用页面画面信息。

### 具体表现
- 更像“带图的 OCR + 文本翻译”，不是整页视觉理解
- 画面中的信息没有被显式用于决策：
  - 构图
  - 角色站位
  - 视线方向
  - 动作幅度
  - 镜头切换
  - 视觉焦点
- 结果会让 internal/direct 的输出比预期更弱，尤其在需要依赖画面语境的页面上

## 影响
- 角色语气、情绪、关系变化的判断可能偏弱
- 复杂分镜页上，翻译会失去画面约束
- 让 internal/direct 链路在基准测试中看起来“理解了文字，却没理解画面”

## 当前假设
1. Vision Stage prompt 没有强制“先整页、后气泡”
2. Page JSON schema 对视觉字段约束不够强
3. QA 没有专门检查“是否只读气泡，忽略画面”

## 建议修复方向
- 强制 Vision Stage 先输出整页层面的视觉理解，再做气泡提取
- 在 Page JSON 中显式增加视觉语境字段
- 在 QA Stage 中增加“视觉语境丢失”检查项

## 验证方法
- 选取几页画面依赖强的样本，比较：
  - 仅看气泡即可翻出的页面
  - 必须依赖画面语境才能正确处理的页面
- 若模型在后者上仍表现得像纯文本翻译，则说明链路仍然退化
